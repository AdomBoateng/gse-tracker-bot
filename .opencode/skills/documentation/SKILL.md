# Documentation

## Purpose
Enable agents to produce clear, structured, and accurate documentation.

## When to Use This Skill
Use this skill when writing API docs, architecture guides, usage instructions, or onboarding materials.


## Rules
1. Keep docs concise and readable.
2. Include code examples where applicable.
3. Use version control to track changes.

## Output Format
Provide:
- Structured documentation in Markdown
- API reference, architecture overview, or usage guide as applicable
- Code examples and diagrams where relevant
- Version and last-updated metadata
