# CI/CD Pipelines

## Purpose
Enable agents to automate development workflows for faster, reliable releases.

## When to Use This Skill
Use this skill when creating or updating build, test, or deployment automation for any service or project.


## Rules
1. Ensure pipelines are modular and maintainable.
2. Avoid manual interventions in automated workflows.
3. Document all pipeline steps and triggers.

## Output Format
Provide:
- Pipeline configuration file (YAML or equivalent)
- Stage-by-stage description of what each step does
- Trigger conditions and rollback strategy
- Required secrets or environment variables
