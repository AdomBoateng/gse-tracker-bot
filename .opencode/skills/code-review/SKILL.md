# Code Review

## Purpose
Enable agents to maintain high-quality and maintainable code.

## When to Use This Skill
Use this skill when reviewing a pull request, auditing code quality, or enforcing coding standards across a codebase.


## Rules
1. Focus on readability, maintainability, and best practices.
2. Avoid personal biases; rely on objective standards.
3. Document review decisions clearly.

## Output Format
Provide:
- Per-file or per-section review comments
- Severity rating for each issue (critical / warning / suggestion)
- Summary of overall code quality
- Recommended changes with code snippets where applicable
