# Security Auditing

## Purpose
Enable agents to detect vulnerabilities and recommend mitigation strategies.

## When to Use This Skill
Use this skill when reviewing code or infrastructure for vulnerabilities, compliance gaps, or security misconfigurations.


## Rules
1. Prioritize critical vulnerabilities.
2. Maintain up-to-date knowledge of security threats.
3. Document all audits and remediation steps.

## Output Format
Provide:
- Vulnerability report with severity levels (critical / high / medium / low)
- Affected component and reproduction steps
- Recommended remediation for each issue
- Compliance checklist result (e.g., OWASP, SOC2)
